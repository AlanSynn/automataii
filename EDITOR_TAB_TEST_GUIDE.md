# 에디터 탭 & 스켈레톤 추출 테스트 가이드

## 🎯 목적
이 가이드는 수정된 에디터 탭 로딩과 스켈레톤 추출 기능을 테스트하기 위한 것입니다.

## ✅ 수정된 기능
1. **스켈레톤 추출 기능** - 빈 플레이스홀더에서 실제 기본 스켈레톤 생성으로 업그레이드
2. **자동 에디터 전환** - 스켈레톤 생성 후 자동으로 에디터 탭으로 전환
3. **오류 허용 처리** - annotation 실패 시에도 기본 스켈레톤 생성

## 🔧 테스트 절차

### Phase 1: 이미지 로딩 테스트
1. **애플리케이션 실행 확인**
   ```bash
   python verification_tracer.py &
   ```

2. **랜딩 탭에서 예시 이미지 선택**
   - astronaut.png 또는 alien.png 클릭
   - ✅ 이미지 처리 탭으로 자동 전환 확인

### Phase 2: 스켈레톤 추출 테스트
3. **이미지 처리 실행**
   - "Process Image" 버튼 클릭
   - ✅ 진행 다이얼로그 표시 확인
   - ✅ 처리 완료 후 기본 스켈레톤 생성 확인

4. **자동 에디터 전환 확인**
   - ✅ 스켈레톤 처리 완료 후 자동으로 "Path Editor" 탭으로 전환
   - ✅ 에디터에 스켈레톤 구조 표시 확인

### Phase 3: 에디터 기능 테스트
5. **스켈레톤 시각화 확인**
   - ✅ 기본 스켈레톤 joints 표시 (root, torso, head, arms, legs)
   - ✅ Joint 간 연결선 표시
   - ✅ 마우스 호버/클릭 반응성 확인

6. **패스 드로잉 테스트**
   - 스켈레톤 부위 선택 (예: arm)
   - ✅ 드로잉 모드 활성화 확인
   - ✅ 마우스로 motion path 그리기 가능 확인

### Phase 4: 애니메이션 테스트
7. **애니메이션 컨트롤 테스트**
   - Play 버튼 클릭
   - ✅ 스켈레톤이 그린 패스를 따라 움직임 확인
   - ✅ Stop/Reset 버튼 기능 확인

## 🐛 예상 문제 및 해결책

### 문제 1: 스켈레톤이 에디터에 표시되지 않음
**해결책:** 
- 스켈레톤 매니저 초기화 확인
- 이벤트 버스 연결 상태 확인

### 문제 2: 이미지 처리 후 에디터로 자동 전환되지 않음
**해결책:**
- `skeleton_updated` 시그널 연결 확인
- `switch_to_editor_tab()` 메서드 호출 확인

### 문제 3: 애니메이션이 작동하지 않음
**해결책:**
- 키네마틱스 시스템 초기화 확인
- IK 애니메이터 설정 확인

## 📊 성공 기준

### ✅ 완전 성공 (모든 기능 작동)
- 이미지 로딩 → 스켈레톤 추출 → 에디터 전환 → 패스 드로잉 → 애니메이션

### ⚠️ 부분 성공 (기본 기능 작동)
- 이미지 로딩 → 스켈레톤 추출 → 에디터 전환 (애니메이션은 별도 수정 필요)

### ❌ 실패 (주요 기능 미작동)
- 스켈레톤 추출 실패 또는 에디터로 전환되지 않음
