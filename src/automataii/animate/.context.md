# Context: src/automataii/animate/

This directory, `animate/`, is a sub-package of `automataii` dedicated to animation functionalities. It includes modules for extracting animatable parts, defining their structure, generating animations from various sources (images, annotations), and potentially applying advanced deformation techniques.

## Core Responsibilities:

- **Part Extraction & Definition**: Identifying and defining animatable components (e.g., body parts of a character) from source material like images.
- **Annotation Processing**: Working with annotations (e.g., skeletal data, control points) to guide animation.
- **Animation Generation**: Creating animation sequences, possibly involving keyframing, interpolation, and procedural motion.
- **Deformation**: Implementing techniques for realistic or stylized deformation of animated parts (e.g., As-Rigid-As-Possible).

## Key Files & Modules:

- **`body_parts_extractor.py`**: Module responsible for algorithms that identify and extract distinct parts from images or other inputs, preparing them for animation.
- **`body_parts_animation.py`**: Core module for animating the extracted and defined body parts. This likely handles transformation (rotation, translation, scale) over time, keyframing, and playback.
- **`part_definitions.py`**: Defines the structure, properties, and relationships of the animatable parts (e.g., hierarchy, joint types, constraints).
- **`image_to_annotations.py`**: Functionality to process images and generate structured annotations (e.g., pose estimation, part segmentation masks) that can drive animation.
- **`image_to_animation.py`**: Modules that directly generate animation data from static images or image sequences, possibly by combining extraction, annotation, and animation logic.
- **`annotations_to_animation.py`**: Converts pre-existing structured annotation data into animation sequences.
- **`templates.py`**: May contain predefined animation cycles, poses, or skeletal templates to streamline the animation process or provide starting points.
- **`arap.py`**: Implements As-Rigid-As-Possible (ARAP) deformation algorithms. This is a common technique for producing natural-looking deformations of 2D or 3D shapes while preserving local rigidity.
- **`__init__.py`** (expected): Makes `animate/` a Python sub-package.

- **Clear Separation**: Maintain a clear separation between part extraction, definition, animation logic, and deformation techniques.
