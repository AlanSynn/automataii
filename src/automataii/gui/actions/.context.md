# Context: src/automataii/gui/actions/

This sub-package, `actions/`, within `src/automataii/gui/`, is responsible for defining, managing, and organizing `QAction` objects for the `automataii` application. `QAction` instances represent user-triggerable commands that can appear in menus, toolbars, or be invoked by keyboard shortcuts.

## Purpose:

- **Centralized Command Definition**: To provide a single place to define all user actions, ensuring consistency in text, icons, shortcuts, tooltips, and behavior.
- **Decoupling**: To decouple the action (the command itself) from the UI elements (menus, buttons) that trigger it. The same `QAction` can be added to multiple UI components.
- **State Management**: Actions can have states (enabled/disabled, checkable/checked) that can be managed centrally, often based on the application's current context or state.

## Key Files & Contents:

- **`action_manager.py`**: This module likely contains an `ActionManager` class or a set of functions responsible for:
    - Creating `QAction` instances.
    - Setting their properties (text, icon, shortcut, status tip, what's this, etc.).
    - Connecting the `triggered` signal of each action to an appropriate slot (handler method), which is often located in `main_window.py` or a relevant controller/presenter class.
    - Organizing actions, perhaps in groups (e.g., file actions, edit actions, view actions).
    - Providing access to these actions for other parts of the GUI (e.g., for building menus and toolbars).

## Integration with PyQt6/Qt:

- `QAction` is a fundamental Qt class.
- An `ActionManager` would typically be instantiated by `main_window.py`.
- The `MainWindow` would then use the manager to retrieve actions and add them to `QMenu` objects (via `menu.addAction(action)`) and `QToolBar` objects (via `toolbar.addAction(action)`).

## Development Guidelines:

- **Naming Conventions**: Use clear and consistent names for action objects (e.g., `self.open_file_action`, `self.save_action`).
- **Icon Management**: Icons for actions should be managed systematically, possibly loaded from a resource file or a dedicated `resources/assets/icons/` directory (as per project guidelines).
- **Shortcuts**: Define standard and intuitive keyboard shortcuts for common actions.
- **Status Tips & Tooltips**: Provide helpful status tips and tooltips for all actions.
- **Contextual Enabling/Disabling**: The `ActionManager` or the `MainWindow` should handle enabling/disabling actions based on the current application state (e.g., disable "Save" if there are no unsaved changes).
- **Modularity (for large applications)**: If the number of actions becomes very large, consider further subdividing this package (e.g., `actions/file_actions.py`, `actions/edit_actions.py`) and having `action_manager.py` aggregate them.
- **Internationalization (i18n)**: Action text (labels, tooltips) should be translatable if the application supports multiple languages.
