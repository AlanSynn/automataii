# Context: src/automataii/gui/widgets/

This sub-package, `widgets/`, within `src/automataii/gui/`, is intended to house custom, reusable UI widgets developed specifically for the `automataii` application. These widgets typically extend standard PyQt6 (or other Qt framework) widget classes or combine multiple standard widgets to create more complex, specialized UI components.

## Purpose:

- **Encapsulation**: To encapsulate specific UI functionalities, appearances, and interactions into self-contained, manageable components.
- **Reusability**: To create widgets that can be easily instantiated and used in multiple parts of the application (e.g., in different dialogs, tabs, or main window layouts), promoting a DRY (Don't Repeat Yourself) approach to UI development.
- **Maintainability & Clarity**: To simplify the code of higher-level UI containers (like `main_window.py` or tab modules) by delegating detailed UI logic and layout to these specialized widget classes.

## Key Files & Contents:

- **`processing_steps_group.py`**: Defines a custom widget, likely a `QGroupBox` or `QFrame` subclass, designed to display or manage a sequence of processing steps or stages. This might involve displaying status, allowing configuration for each step, or visualizing progress.
- **`__init__.py`**: Makes `widgets/` a Python sub-package. It should export the custom widget classes (e.g., `ProcessingStepsGroup`) for easy import and use by other GUI modules.
- **`__pycache__/`**: Standard Python bytecode cache directory (will be skipped as per instruction).

## Development Guidelines for Custom Widgets:

- **Base Class**: Custom widgets SHOULD inherit from an appropriate Qt base class (e.g., `QWidget`, `QFrame`, `QGroupBox`, or even more specialized ones like `QPushButton` if creating a custom button type).
- **API Design**: Design widgets with a clear and well-defined API:
    - **Properties**: Use Qt properties (`pyqtProperty`) for attributes that need to be configurable or accessible from Qt Designer or QSS.
    - **Methods**: Provide methods for controlling behavior or updating content.
    - **Signals**: Define custom signals (`pyqtSignal`) to communicate events or data changes from the widget to the outside world (e.g., a value changed, an action was triggered).
    - **Slots**: Implement slots (`pyqtSlot`) to allow the widget to respond to signals from other objects or its own internal components.
- **Self-Contained**: Widgets should manage their own internal state, layout, and child widgets as much as possible.
- **Layout**: Use Qt layout managers (`QVBoxLayout`, `QHBoxLayout`, `QGridLayout`, `QFormLayout`) internally to arrange child widgets. Avoid fixed-size geometries where possible to allow for resizability and adaptability.
- **Styling**: Widgets should be stylable using Qt Style Sheets (QSS). Consider defining custom QSS properties if needed.
- **File Organization**: Each distinct custom widget should ideally reside in its own Python file within this directory (e.g., `my_custom_slider.py`).
- **Documentation**: Provide clear docstrings for the widget class and its public API (methods, signals, properties).
