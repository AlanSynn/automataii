# Context: src/automataii/gui/graphics_items/

This sub-package, `graphics_items/`, within `src/automataii/gui/`, is dedicated to custom subclasses of `QGraphicsItem` (and related classes) from Qt's Graphics View Framework. These items are visual elements that can be added to a `QGraphicsScene` and displayed via a `QGraphicsView`, enabling rich 2D graphical interactions.

## Purpose:

- **Custom 2D Graphics**: To render and manage custom 2D graphical objects that go beyond standard widgets. This is suitable for diagrams, interactive visualizations, custom drawing tools, or game-like scenes.
- **Interactivity**: To create items that respond to mouse events (clicks, drags, hover), keyboard input, and can be selected, moved, resized, or rotated within the scene.
- **Scene Management**: To organize complex visual elements in a structured way, allowing for layering, grouping, and efficient updates.

## Key Files & Contents:

- **`part_item.py`**: Defines a `QGraphicsItem` representing a "part" or component within a graphical scene. This could be an element of a character (if used for animation visualization), a piece in a diagram, or any distinct visual entity.
    - Likely handles its own painting, bounding box, and interaction logic.
- **`skeleton_item.py`**: Defines a `QGraphicsItem` for visualizing a skeleton or a hierarchical structure (e.g., bones in a character rig, nodes in a tree). It might draw lines/bones connecting joints or parts.
- **`anchor_item.py`**: Represents an anchor point, control point, or handle within the scene. These are often small, interactive items used to manipulate other `QGraphicsItem`s (e.g., resizing handles, Bezier curve control points, joint positions).
- **`__init__.py`**: Makes `graphics_items/` a Python sub-package. It should export the custom `QGraphicsItem` classes (e.g., `PartItem`, `SkeletonItem`, `AnchorItem`) for easy import and use by the modules that manage the `QGraphicsScene` (likely in `main_window.py` or a dedicated view class).
- **`__pycache__/`**: Standard Python bytecode cache directory for this sub-package.

## Core Qt Graphics View Concepts Used:

- **`QGraphicsItem`**: The base class for all items in a `QGraphicsScene`. Key methods to override include:
    - `boundingRect()`: Returns the outer bounds of the item in local coordinates. Essential for drawing and collision detection.
    - `paint(QPainter, QStyleOptionGraphicsItem, QWidget)`: Renders the item's content.
    - Event handlers: `mousePressEvent()`, `mouseMoveEvent()`, `hoverEnterEvent()`, etc.
- **`QGraphicsScene`**: Manages all the `QGraphicsItem`s. Provides functions for adding, removing, and finding items.
- **`QGraphicsView`**: The widget used to display the `QGraphicsScene` and provides a viewport onto the scene, handling scrolling and transformations.
- **Coordinate Systems**: Item coordinates, Scene coordinates, and View coordinates.

## Development Guidelines:

- **Ownership**: Each `QGraphicsItem` is responsible for its own appearance and basic interaction logic.
- **Performance**: Be mindful of painting efficiency, especially with many items or complex rendering. Use `QGraphicsItem.ItemSendsGeometryChanges` and `prepareGeometryChange()` where appropriate. Avoid unnecessary redraws.
- **Hit Testing**: Ensure `shape()` is overridden if the item has a non-rectangular shape for accurate mouse interaction.
- **Modularity**: Keep item classes focused on their specific visual representation and behavior.
- **Data Binding**: If items represent underlying data from `src/automataii/core/` or `src/automataii/models/`, ensure clear mechanisms for updating the visual item when the data changes (and vice-versa if items are editable).
