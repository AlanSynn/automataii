# Context: src/automataii/gui/

This directory, `gui/`, is the primary sub-package within `automataii` responsible for all aspects of the Graphical User Interface (GUI). It follows a structured organization, common in PyQt6 (or similar Qt-based) applications, to manage windows, widgets, actions, dialogs, and other UI elements.

## Core Responsibilities:

- **Main Application Window**: Defining and orchestrating the primary window(s) of the application.
- **UI Components**: Implementing custom widgets, dialogs, and specialized views.
- **User Interaction**: Handling user input through actions (menus, toolbars), events, and interactive graphics.
- **Visual Presentation**: Managing the look and feel, including layouts, styling (QSS), and custom graphics items.

## Key Files & Subdirectories:

- **`main_window.py`**: The central `QMainWindow` (or equivalent) subclass that serves as the main application frame. It initializes and arranges toolbars, menubars, status bars, and central widgets/views.
- **`actions/`**: Sub-package for defining and managing `QAction` objects. Actions represent commands triggered by menus, toolbars, or shortcuts. See `src/automataii/gui/actions/.context.md`.
- **`dialogs/`**: Sub-package containing custom `QDialog` subclasses for specific user interactions like settings, confirmations, or input forms. See `src/automataii/gui/dialogs/.context.md`.
- **`fonts/`**: Directory for storing custom font files used by the GUI. See `src/automataii/gui/fonts/.context.md`.
- **`graphics_items/`**: Sub-package for custom `QGraphicsItem` subclasses used within a `QGraphicsScene` for advanced 2D graphics. See `src/automataii/gui/graphics_items/.context.md`.
- **`tabs/`**: Sub-package for widgets that are used as individual pages within a `QTabWidget` or similar tabbed interface. See `src/automataii/gui/tabs/.context.md`.
- **`views/`**: Sub-package for views that are used by windows and tabs. See `src/automataii/gui/views/.context.md`.
- **`widgets/`**: Sub-package for custom, reusable UI widgets that extend or combine standard Qt widgets. See `src/automataii/gui/widgets/.context.md`.
- **`__init__.py`** (expected): Makes `gui/` a Python sub-package.
- **`__pycache__/`**: Standard Python bytecode cache for this sub-package.

## Design Philosophy (likely based on PyQt6/Qt):

- **Model-View-Controller (MVC) or Model-View-Presenter (MVP)**: While not strictly enforced by file structure alone, Qt development often encourages separating data (model), presentation (view), and logic (controller/presenter).
- **Signal-Slot Mechanism**: Core to Qt for communication between objects, enabling loose coupling.
- **Modular Widgets**: Creating self-contained, reusable widgets is a key principle.
- **Layout Management**: Using Qt's layout classes (`QVBoxLayout`, `QHBoxLayout`, `QGridLayout`, etc.) for responsive and maintainable UI arrangements.

## Interactions with Other `automataii` Modules:

- **`core/`**: The GUI will interact with core logic to display data and trigger operations.
- **`config/`**: To load and potentially save UI-related settings.
- **`animate/`, `kinematics/`, etc.**: The GUI may provide interfaces to control or visualize functionalities from these modules.
- **`models/`**: To display and interact with application-specific data structures.

## Development Guidelines:

- Adhere to the PyQt6 best practices and the project-specific UI structure guidelines (if any are formalized beyond this file structure).
- Keep UI logic separate from business logic.
- Ensure custom widgets and dialogs are well-documented and reusable where possible.
- Use Qt Designer (`.ui` files) for layout if preferred, and `uic.loadUi` or code generation for integration, or define layouts purely in Python code.
