# Context: src/automataii/config/

This sub-package, `config/`, within `automataii`, is designated for managing application-wide configuration settings, parameters, and constants.

## Purpose:

- **Centralized Configuration**: To provide a single, organized location for all configurable aspects of the `automataii` application.
- **Separation of Concerns**: To separate configuration data from application logic, making it easier to modify settings without changing code.
- **Parameter Management**: To define and provide access to parameters that control the behavior of various modules (e.g., `gui`, `animate`, `kinematics`).

## Key Files & Contents:

- **`z_indices.py`**: This module likely defines Z-order (or stacking order) constants, primarily for graphical elements within the GUI. Z-indices determine which widgets or items are drawn on top of others when they overlap.
    - Example constants might include `TOP_LAYER = 10`, `MIDDLE_LAYER = 5`, `BOTTOM_LAYER = 0`.
- **`__init__.py`** (expected, may be currently missing or empty): Makes `config/` a Python sub-package. It might export key configuration classes, constants, or functions for loading configurations.
- **`__pycache__/`**: Standard Python bytecode cache (will be skipped).

This directory could be expanded to include:

- **`settings.py` or `app_config.py`**: A module defining a Pydantic `BaseSettings` class or a custom configuration class to load settings from environment variables, `.env` files, or configuration files (e.g., `config.ini`, `config.yaml` at the project root).

- Modules for specific configuration areas: `gui_config.py`, `logging_config.py`, `api_keys.py` (though sensitive keys should ideally be managed via environment variables or a secrets manager).

## Integration & Usage:

- Modules throughout the `automataii` application (e.g., in `gui`, `core`, `animate`) would import necessary configuration values or objects from this `config` package.
    - Example: `from automataii.config.z_indices import TOOLTIP_Z`
    - Example: `from automataii.config import settings; API_URL = settings.API_URL`
- Configuration loading (if applicable from files or environment) should ideally happen once at application startup.

## Development Guidelines:

- **Clarity and Organization**: Group related configuration parameters logically, either within the same file or in separate, aptly named files.
- **Type Hinting**: Use type hints for configuration values, especially if using Pydantic for settings management.
- **Defaults**: Provide sensible default values for all configuration parameters.
- **Documentation**: Clearly document what each configuration parameter does and its possible values or format.
- **Avoid Hardcoding**: Minimize hardcoded values in other modules; instead, define them here and import them where needed.
- **Security**: For sensitive information (API keys, passwords), use environment variables or a dedicated secrets management solution. Do not hardcode them or commit them to version control, even in configuration files (use `.env` files excluded by `.gitignore`).
