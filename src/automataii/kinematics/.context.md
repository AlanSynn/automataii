# Context: src/automataii/kinematics/

This sub-package, `kinematics/`, within `automataii`, is dedicated to modules related to kinematics. This primarily involves the study and computation of motion for articulated systems (e.g., robotic arms, character limbs) without considering the forces that cause it. Inverse Kinematics (IK) and Forward Kinematics (FK) are central concepts here.

## Core Responsibilities:

- **Inverse Kinematics (IK)**: Calculating the joint parameters (angles, positions) required for an end-effector (e.g., a hand or tool) to reach a desired position and orientation in space.
- **Forward Kinematics (FK)**: Calculating the end-effector's position and orientation given a set of joint parameters.
- **Kinematic Chain Management**: Defining and managing kinematic chains (sequences of links and joints).
- **Configuration**: Handling parameters specific to kinematic models, such as link lengths, joint limits, and solver settings.
- **Utilities**: Providing mathematical utilities for transformations, vector math, and quaternion operations relevant to kinematics.

## Key Files & Subdirectories:

- **`ik_manager.py`**: A central module or class (`IKManager`) responsible for orchestrating IK calculations. It likely:
    - Takes a kinematic chain definition and a target pose as input.
    - Interfaces with one or more IK solver algorithms (from the `solvers/` subdirectory).
    - Returns the resulting joint configurations.
- **`kinematic_config.py`**: Defines data structures or classes for holding kinematic configuration parameters. This might include parameters for different robot models, joint constraints (min/max angles), link dimensions, and solver-specific settings.
- **`solvers/`**: A sub-package containing different implementations of IK solver algorithms. Each solver might use a different mathematical approach (e.g., Jacobian Inverse, FABRIK, CCD, optimization-based). See `src/automataii/kinematics/solvers/.context.md`.
- **`utils.py`**: Contains utility functions specifically for kinematic calculations. This could include functions for 3D transformations, matrix operations, vector math, quaternion conversions, or angle normalization.
- **`test_ik_manager.py`**: Contains unit tests for `ik_manager.py` and potentially other core kinematics functionalities, ensuring correctness of the IK solutions and configurations.
- **`__init__.py`**: Makes `kinematics/` a Python sub-package. It should export key classes and functions like `IKManager` and perhaps an interface to the solvers or configuration loaders.
- **`__pycache__/`**: Standard Python bytecode cache (will be skipped).

## Interactions with Other `automataii` Modules:

- **`models/`**: May use data models from `src/automataii/models/` to represent poses, joint states, or kinematic chain structures.
- **`animate/`**: The animation system (`src/automataii/animate/`) is a primary consumer of this kinematics module, using IK to pose characters or robotic elements.
- **`gui/`**: The GUI might use kinematics to allow interactive posing of models or to display kinematic simulations.
- **`core/`**: May rely on core mathematical constructs or data structures.

## Development Guidelines:

- **Accuracy and Performance**: IK solvers can be computationally intensive. Prioritize both the accuracy of the solutions and the performance of the calculations.
- **Solver Abstraction**: The `IKManager` should provide a clean interface to different solvers, allowing them to be swapped or selected based on needs.
- **Clear Configuration**: Kinematic configurations (`kinematic_config.py`) should be well-structured, easy to understand, and modifiable.
- **Robustness**: Handle edge cases such as unreachable targets, singularities, and joint limit violations gracefully.
- **Testing**: Thoroughly test IK solutions against known configurations and for various target poses and constraints.
