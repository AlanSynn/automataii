# Context: src/automataii/kinematics/solvers/

This sub-package, `solvers/`, within `src/automataii/kinematics/`, is specifically dedicated to housing different implementations of Inverse Kinematics (IK) solver algorithms. Each module in this directory typically provides one or more distinct approaches to solving the IK problem.

## Purpose:

- **Algorithmic Variety**: To provide a collection of different IK algorithms. Different solvers have various trade-offs in terms of computational cost, accuracy, stability, convergence properties, and suitability for specific types of kinematic chains or constraints.
- **Modularity**: To encapsulate each solving technique into its own module, making it easier to manage, test, and select a solver within the `IKManager`.

## Key Files & Contents:

- **`base_ik_solver.py`**: Defines an abstract base class (ABC) or a common interface (protocol) that all concrete IK solver implementations in this directory MUST adhere to. This ensures that the `IKManager` (from the parent `kinematics` package) can interact with any solver in a standardized way. The base class would typically define methods like `solve(target_pose, initial_guess)` and properties for solver-specific parameters.
- **`geometric_ik_solver.py`**: An implementation of an IK solver based on geometric principles. Geometric solvers are often very fast and provide exact solutions for specific, simpler kinematic configurations (e.g., 2-link or 3-link planar arms, spherical wrists). They might not be suitable for highly complex or redundant manipulators.
- **`__init__.py`**: Makes `solvers/` a Python sub-package. It should export the concrete solver classes (e.g., `GeometricIKSolver`) and potentially the `BaseIKSolver` interface for type hinting and use by `IKManager`.
- **`__pycache__/`**: Standard Python bytecode cache (will be skipped).

## Potential Additional Solvers (Illustrative):

This directory might be extended with other common IK solver types:
- `jacobian_ik_solver.py`: Using Jacobian matrix methods (e.g., Jacobian Transpose, Damped Least Squares).
- `fabrik_solver.py`: Forward And Backward Reaching Inverse Kinematics (FABRIK), an iterative heuristic solver.
- `ccd_solver.py`: Cyclic Coordinate Descent, another iterative heuristic solver.
- `optimization_ik_solver.py`: Solvers that frame IK as an optimization problem, using numerical optimization techniques.

## Development Guidelines:

- **Interface Compliance**: All new solver implementations MUST strictly adhere to the interface defined in `base_ik_solver.py`.
- **Documentation**: Each solver module should thoroughly document:
    - The algorithm used.
    - Its assumptions and limitations.
    - Computational complexity.
    - Any specific parameters it requires.
- **Testing**: Each solver should have dedicated unit tests to verify its correctness, convergence, and behavior under various conditions (e.g., reachable targets, unreachable targets, singularities, joint limits).
- **Performance Profiling**: For computationally intensive solvers, profile their performance and optimize where possible.
- **Parameterization**: Allow solver-specific parameters (e.g., iteration limits, tolerance) to be configured, possibly via the `kinematic_config.py` from the parent package.
